# Security Policy

## Supported Releases

Only the latest tagged release is considered supported.

## Reporting A Vulnerability

Do not open public issues for security reports.

Use one of these channels:
- GitHub private vulnerability reporting, if enabled on the repository
- direct contact with the repository maintainer

When reporting, include:
- affected version
- reproduction steps
- impact summary
- suggested mitigation, if known

We will assess, reproduce, and patch in a new release tag.
