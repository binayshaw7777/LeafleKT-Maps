# Android Compose Playground

Purpose:
- Build sample/demo screens that make library options explorable.

When to use:
- Adding controls, previews, presets, sample data, and QA surfaces in `:app`

Workflow:
1. Expose configuration as editable state.
2. Keep controls grouped and deterministic.
3. Prefer presets plus fine-grained controls.
4. Ensure the preview surface reflects the exact library API.

Outputs:
- Sample screen implementation
- Preview scenarios
- Manual QA notes
